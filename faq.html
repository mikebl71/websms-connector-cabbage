    What is "Cabbage Gateway"?

    It is a web site that can take your details like your phone number and message, login on your behalf to the web site of your mobile provider and send your message as a free web text. The great thing is that, if you do this from your mobile, the data traffic between your mobile and the Gateway is kept to an absolute minimum. All the verbose talking is done between the gateway and the provider's web site. Which means that if you pay for your mobile data per kilobyte then the extra cost per message is minimal. You can read more about it here.

    What is "WebSMS Connector"?

    WebSMS is an android app that was developed to send free/cheep sms messages. The great power of this app comes from its plugin architecture. It allows you to add support for more and more providers by installing new Connectors. So, if you install the "Fishtext Connector", you can send messages via Fishtext.com, and if you install this Cabbage Connector, you can send messages via Cabbage Gateway, etc. You can read more about it here.

    Which mobile providers are supported?

    Irish mobile providers: Vodafone, Meteor, Three, Tesco Mobile, eMobile, and also O2 UK and Telenor Serbia. This application sends sms using web sites of these mobile providers (via Cabbage Gateway). So you must be registered on one of these web sites (have username and password to login) in order to use this application.

    Why "O2" is not in the list of supported mobile providers?

    Unfortunately, O2 decided to block the site where Cabbage Gateway is running. So the default Cabbage Gateway does not currently work with O2.

    However, if you know a bit about computers then there is a workaround. You can install a copy of Cabbage Gateway on your own web site. Then, provided that you don't share it with too many people and stay under the O2's radar, you can use it with no problems. This answer describes how to switch to your own gateway. Then, since "O2" is not in the official Providers list, you need to use the "Custom" provider. When you add your account, choose "Custom" in the "Provider" list and type "o" (without quotes - cabbage code for O2) in the "Custom Provider" field.

    How much does it cost to send a message?

    This largely depends on your mobile provider. With this way of sending messages, you don't pay for sending sms, you pay for mobile data that was used while talking to the Gateway. An average message will use about 0.1 kilobyte, give or take. An average price for mobile data in Ireland is about 1c per 1 kilobyte. In such case, the message will cost you around 0.1c. However, this varies wildly between providers. Some give free data for a month after a topup, some charge per kilobyte at various rates, some charge minimum price per day, some offer various flat rates. And if you are roaming then it's a different price again. So do check the prices with your mobile provider.

    Also, "free" web texts are not always entirely free. They usually have some conditions like national destinations only, or for one month after topup only. And you might be charged full sms rate you you send a message outside these boundaries. So do read the Terms and Conditions of your web texts.

    If you are confused then the simplest way to figure this out is to send a few test messages (and check your balance for the first while). Then you'll know for sure how much it costs and whether it's worth it.

    Does this app support international destinations, non-english characters?

    This entirely depends on whether the web texts of your mobile provider support these. If their web site doesn't say anything explicitly, then the best way to verify this is to login to their web site, try to send a test message to your destination in your language and see if it arrives correctly. If it does then it will most likely work if sent via this application as well.

    I am registered with several Irish mobile providers and want to switch between them, is this supported?

    Yes, you can add all your accounts to this Connector and choose which to use when you send the message. Note though that the message will look as if sent from the chosen account (and the respective phone number). So, if your recipient will try to reply back, the reply will go to that number.

    Is it safe to use Cabbage Gateway?

    Well, that depends. The way Cabbage Gateway works is that it needs to know your username, password and message text in order to send it on your behalf from the mobile provider's web site. So you do need to trust Cabbage developer that he will not try to access this information. However, this gateway is used by loads of people in Ireland and, so far, no-one accused him of any wrong doing. So, I'd say, unless you send extremely sensitive information (which sms are no good for anyway) then you'll be fine. Additionally, if you want to play safe, you can actually install your own copy of the gateway as described here.

    Can I install my own copy of Cabbage Gateway

    Absolutely. Cabbage Gateway is just a bunch of PHP scripts. So, if you know of a good PHP hosting site (it must supports "curl"), then you can download the scripts from here and deploy them on your site. After that, go to the preferences screen of WebSMS Cabbage Connector, un-tick "Default Cabbage Gateway" and type in the url of your scripts into the "Custom Cabbage Gateway" field (for example, "http://yourhost.com/cabbage/send.php", without quotes).

    Could you support provider X?

    A provider must be supported by the Cabbage Gateway first. So please contact Cabbage developer here and ask him.

    Alternatively, if you are a developer yourself, you can develop your own script in a similar format and install it on some web site. As soon as this script has the same interface as Cabbage, you can use this Connector to access it. Have a look at this and this answers.

    What if I know that provider Y is supported by Cabbage Gateway but somehow it is not in the Providers list?

    First of all, let me know and I'll add it to the Providers list. But in meanwhile, you can select "Custom" in the "Provider" list and then type in the provider's code in the "Custom Provider" field. Note that this has to be the internal Cabbage's provider code like "o" for "O2" or "te" for "Tesco Mobile".

    For Tesco Mobile, why does it show international balance only?

    Tesco Mobile is a bit special in the sense that they have two balances: one for national and another for international messages. Neither WebSMS nor Cabbage Gateway support this. So, at present, Cabbage Gateway will return balance for the last message sent. Alternatively, if you press "Update balance", it will return the international balance only. I would think that something like the minimum of the two balances would be a better compromise. I asked the Cabbage developer about this but so far it remains the same. You can ask him as well to keep up the pressure.

    I added two accounts to the Connector but only the first balance is shown, why?

    This is a known issue. Unfortunately, this is not that easy to fix as it requires changes to the WebSMS framework. But I am working on this.
